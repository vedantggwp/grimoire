## Build
- `scripts/build.mjs` ‚Äî esbuild production bundler; produces three self-contained ESM bundles in `dist/` with the mandatory `createRequire`/`__dirname`/`__filename` ESM banner (per esbuild issue #1921); target `node20`, `packages: bundle`, `sourcemap: linked`, `minifyWhitespace: true`; includes the `d3-inline` onLoad plugin that swaps `lib/present/modes/d3-source.ts` for an inlined `d3.min.js` string at build time
- `scripts/mcp-smoke-client.mjs` ‚Äî Protocol-level MCP smoke test client: spawns `dist/serve.js` against `grimoire-wiki`, exercises all 7 tools via real JSON-RPC over stdio using `@modelcontextprotocol/sdk`'s stdio client. Same transport Claude Desktop uses; clean pass here = clean pass in any MCP client. Runnable at any time: `node scripts/mcp-smoke-client.mjs`
- `dist/compile.js` ‚Äî Bundled compile CLI (~661 KB) + `compile.js.map` linked sourcemap; shipped for marketplace install (regenerated by `npm run build`)
- `dist/present.js` ‚Äî Bundled present CLI (~949 KB, includes inlined d3.min.js for self-contained graph mode) + sourcemap
- `dist/serve.js` ‚Äî Bundled serve CLI (~920 KB, largest because it inlines MCP SDK + zod) + sourcemap

## Lib ‚Äî Runtime Scripts (source)
- `lib/short-topic.ts` ‚Äî Pure utility for extracting a concise display name from the verbose SCHEMA.md topic field (delimiter-aware, 60-char cap). Shared between `lib/present/hub.ts` (hub title, page titles, nav brand) and `lib/serve.ts` (MCP server instructions, query results, list_topics header, startup log) so verbose multi-line topics never bleed into display surfaces
- `lib/compile.ts` ‚Äî Papyr Core orchestration: graph analysis, link validation, search indexing, analytics. CLI accepts either a grimoire workspace root (auto-detects `wiki/` subdirectory via `statSync`) or a wiki directory directly
- `lib/present/index.ts` ‚Äî Static site generator entry point, orchestrates all modes
- `lib/present/types.ts` ‚Äî Shared TypeScript types for present modules
- `lib/present/config.ts` ‚Äî Design config parser and palette resolution; default palette: linear-editorial (Source Serif 4 + Inter + JetBrains Mono)
- `lib/present/css.ts` ‚Äî CSS generator: Option F Linear Editorial design system (dual-theme, bento grid, 3-col read, command palette search, timeline feed, treemap gaps, flashcard quiz)
- `lib/present/css-modes.ts` ‚Äî Minimal mode-specific CSS overrides (read progress bar, graph SVG sizing, search results spacing)
- `lib/present/data.ts` ‚Äî Wiki data loader and graph builder
- `lib/present/html.ts` ‚Äî HTML template helpers and page shell
- `lib/present/modes/read.ts` ‚Äî Read mode: article renderer
- `lib/present/modes/graph.ts` ‚Äî Graph mode: interactive knowledge graph with inlined d3 (no CDN)
- `lib/present/modes/d3-source.ts` ‚Äî Exports `d3.min.js` contents as a string; `readFileSync` at dev time, replaced with an inlined literal at build time via the `d3-inline` esbuild plugin
- `lib/present/modes/search.ts` ‚Äî Search mode: full-text search interface
- `lib/present/modes/feed.ts` ‚Äî Feed mode: Option F vertical timeline with left-dated entries, inferred action tags (scouted/compiled/ingested/edited)
- `lib/present/modes/gaps.ts` ‚Äî Gaps mode: Option F treemap grid with 4-tier classification (full/partial/thin/missing), pulsing missing cells, coverage percentages
- `lib/present/modes/quiz.ts` ‚Äî Quiz mode: Option F Duolingo-style 3D buttons, progress bar, flashcard 3D flip (extractSentences handles bullet-list content without terminal punctuation)
- `lib/templates.ts` ‚Äî Template inheritance for the `--from <grimoire>` flag: `loadTemplate(path)` reads `_config/design.md` + `SCHEMA.md` to extract palette/typography/density/motion/audience/level/taxonomyStyle; `applyTemplate` merges template over defaults immutably; 12 tests in `test/templates.test.ts`
- `lib/pipeline-types.ts` ‚Äî Shared TypeScript types for the pipeline orchestration layer
- `lib/defaults.ts` ‚Äî Topic inference module: `inferSchemaFromPrompt(prompt)` returns structured `SchemaDefaults` from a natural-language string (audience/level classification, scope generation, safe defaults for taxonomy/palette/typography); `detectClaudeMd(cwd)` checks for CLAUDE.md presence
- `lib/serve.ts` ‚Äî MCP server exposing wiki as queryable knowledge engine via 7 tools (v0.2.2: added `grimoire_get_section` for section-level addressing, `grimoire_get_article` token guard with `mode: auto|summary|full`, hybrid query reranking that unions FlexSearch + substring search and bonuses title/summary matches, graceful fallback to substring search when FlexSearch export fails, `parseSchemamd` rewritten for canonical nested SCHEMA.md format, summary field carried through every response shape for Karpathy-style LLM routing)
