## Lib ‚Äî Runtime Scripts (source)
- `lib/slug.ts` ‚Äî Single source of truth for slug derivation (basename + slugify lower/strict, exactly papyr-core's rule); imported by `lib/compile.ts` and `lib/present/data.ts` so the definition can't fork
- `lib/update-policy.ts` ‚Äî Parses `_config/update.md` (zod-validated frontmatter + Watchlist/Connection-exclusions body sections) into an `UpdatePolicy`; conservative `DEFAULT_UPDATE_POLICY` when the file is absent ‚Äî the policy file is the human's standing judgment for headless update runs
- `lib/source-ledger.ts` ‚Äî Cross-run dedup ledger: `normalizeUrl` (lowercase host, strip www/fragment/utm/trailing-slash), `normalizeFrontmatterDate` (string or YAML Date), `buildSourceLedger` extracting known URLs from approved-sources.md (any table shape) + raw/ frontmatter, deriving `lastUpdate` from collected dates and log headers ‚Äî no new state file
- `lib/freshness.ts` ‚Äî Per-article staleness tiers (fresh/aging/stale/evergreen/unknown) against policy windows; `checked:` beats `updated:`; dateless articles are `unknown`, never spammed as stale; emits the freshness.json contract consumed by present and serve
- `lib/connections.ts` ‚Äî Deterministic connection-mining candidates: unlinked pairs with ‚â•2 shared tags, ‚â•1 shared (normalized) source, or same-component + shared tag; exclusion-filtered, scored, capped at 20 for bounded LLM judgment cost
- `lib/short-topic.ts` ‚Äî Pure utility for extracting a concise display name from the verbose SCHEMA.md topic field (delimiter-aware, 60-char cap). Shared between `lib/present/hub.ts` (hub title, page titles, nav brand) and `lib/serve.ts` (MCP server instructions, query results, list_topics header, startup log) so verbose multi-line topics never bleed into display surfaces
- `lib/support-slugs.ts` ‚Äî Single source of truth for the structural-page slug set (`index`/`log`/`overview`) and their `.md` filenames; imported by `lib/compile.ts` (enforcement filters) and `lib/present/data.ts` (loader + graph filters) so the definition can't drift between modules
- `lib/compile.ts` ‚Äî Papyr Core orchestration: graph analysis, link validation, search indexing, analytics. CLI accepts either a grimoire workspace root (auto-detects `wiki/` subdirectory via `statSync`) or a wiki directory directly. v0.3.1 adds enforcement-artifact emitters: `overview-metadata.json` every run (top-centrality slugs, required citations, coverage stats, topic clusters) and `taxonomy-proposal.json` conditionally (5+ tags, 5+ content articles, SCHEMA taxonomy not `"defined"`) with deterministic tag-cooccurrence groupings
- `lib/claim-audit.ts` ‚Äî Claim-entailment audit (#48). Extracts statistics + quoted spans from compiled article prose and checks each against the article's cited raw captures; emits `claim-audit.json` via `lib/compile.ts` Step 9.5. Verdicts: supported/unsupported/review/unverifiable, with a sibling-presence heuristic separating fabricated figures from excerpt-trimmed ones. The compile-side half of the G1 provenance guarantee.

Four verdicts: `supported` (present in a capture), `unsupported` (absent from a `full`-fidelity capture ‚Äî decisive defect), `review` (a statistic absent from an *excerpt* whose sibling figures ARE archived ‚Äî the ACON "95%+" pattern, human spot-check), `unverifiable` (no decisive check possible).
