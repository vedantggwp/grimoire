## Core Design Philosophy

"Your agents are only as good as your tools." Unlike traditional integration patterns where middleware orchestrates workflows, agent-based tools must be self-explanatory because LLMs autonomously decide which tools to call and how to interpret results.

## Classification Framework

Tools exist across three dimensions:

1. Maturity: Atomic (single operation) to Orchestrated (coordinated workflows)
2. Integration Type: APIs, databases, file systems, or system operations
3. Access Pattern: Synchronous, asynchronous, streaming, or event-driven

## Four Cross-Cutting Concerns

### Agent Experience
Design descriptions, parameter names, and error messages for LLM comprehension, not human users. The Parameter Coercion pattern: accept variations like "2024-01-15," "January 15," or "yesterday" and normalize internally.

### Security Boundaries
"Prompts express intent, code enforces rules." The Context Injection pattern passes user identity, permissions, and credentials through server-side context, never through the LLM prompt.

### Error-Guided Recovery
Provide actionable guidance instead of raw errors. Rather than returning "429," respond with context: "rate limited, retry after 30 seconds or reduce batch size to 50."

### Tool Composition
Design for consistency across tools — uniform response shapes, batch support to avoid one-at-a-time looping, and multiple abstraction levels for flexible tool selection.

## The 10 Pattern Categories

| Category | Focus |
|----------|-------|
| Tool Types | Query, Command, or Discovery operations |
| Tool Interface | How agents understand and invoke |
| Tool Discovery | Agent-accessible tool identification |
| Tool Composition | Bundling vs. atomic operations |
| Tool Execution | Sync, async, or transactional handling |
| Tool Response | Result formatting for agent consumption |
| Tool Context | Identity and state management |
| Tool Resilience | Failure recovery mechanisms |
| Tool Security | Access control implementation |
| Integration | Connection to external systems |

## Key Distinction from Traditional Integration

The paradigm shift from 2003-style middleware to 2025+ agent tools involves non-deterministic routing (agents choose which tool), stateful sessions replacing stateless messages, and agent-optimized documentation rather than human-readable specifications.
