# Less is More: 4 Design Patterns for Building Better MCP Servers

## Core Principle

"Minimizing context window usage while maximizing relevant information delivery" is crucial for effective MCP design. Current tools already consume significant context — Cursor and Claude Code expose 18 and 15 tools respectively, consuming "5.9% of context window before any user prompt" begins.

## The Four Design Patterns

### 1. Semantic Search

Build a vector similarity search layer above your tool catalog, dynamically retrieving relevant tools based on user queries rather than exposing all tools upfront. Use sentence transformers to encode tool descriptions and queries. Return top-k relevant tools based on similarity scores.

Best for: Large tool inventories with distinct, well-defined purposes.

### 2. Workflow-Based Design

Rather than exposing individual API calls, create single atomic operations handling complete workflows internally. This approach treats "MCP tools as tailored toolkits that help an AI achieve a particular task, not as API mirrors."

Example: Instead of separate create_project(), add_environment_variables(), create_deployment(), add_domain() tools, provide one deploy_project(repo_url, domain, environment_variables, branch="main") tool.

Best for: Well-defined, repeated workflows with predictable steps.

### 3. Code Mode

Pioneered by Cloudflare, this paradigm shifts AI interaction from sequential tool calls to complete program generation executing in secure sandboxes. Real-world improvement: CRM tools previously made 50+ sequential fetch_leads calls with pagination, consuming 200K+ tokens.

Expose a single execute_code tool accepting Python code. The LLM generates optimized code with parallel fetching instead of sequential calls.

Best for: Data processing, batch operations, complex multi-step workflows.

### 4. Progressive Discovery

Guide AI agents through intelligent discovery stages, structuring tool access like human exploration:

1. discover_server_categories_or_actions: Agent identifies relevant services
2. get_category_actions: Agent selects categories and receives action names/descriptions (not full schemas)
3. get_action_details: Full parameter schemas retrieved only when needed
4. execute_action: Perform actual operation

Best for: Platforms with large tool inventories, multi-app workflows.

## Key Takeaway

The "Less is More" principle is not about limiting AI capabilities, but strategically managing context to maximize what agents can do reliably.
