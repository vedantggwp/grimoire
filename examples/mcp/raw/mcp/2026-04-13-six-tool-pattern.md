# The Six-Tool Pattern: MCP Server Design That Scales

> Raw source preserved verbatim. Do not edit this file.
> Compiled into wiki articles during the ingest stage.

---

The six-tool pattern provides a structured approach to MCP server design that balances AI agent control with context efficiency. Rather than creating 15-20 tools with redundant functionality, this pattern consolidates operations into six purpose-driven tools that minimize token waste while maintaining full capabilities.

## The Six Tools

**Category 1: Universal Discovery (OpenAI-Compatible)**

1. **Fetch** — Retrieves specific items using encoded ID strings. The server handles routing complexity internally while the AI uses simple string parameters.

2. **Search** — Accepts query strings supporting both natural language and structured syntax. The server parses the input to determine semantic search versus structured filtering.

**Category 2: Domain-Specific Browsing**

3. **List Collections** — Displays available data sources with optional parameters including schema information, object counts, vectorizer filtering, pagination controls, and name pattern matching.

4. **List Objects** — Browses actual data with parameter-driven control over property selection, pagination, filtering, sorting, and vector inclusion.

**Category 3: Write Operations**

5. **Upsert** — Unified create/update tool routing based on ID presence. Handles single objects, batches, and mixed operations.

6. **Delete** — Removes items via type parameter specifying object deletion, batch deletion, or collection removal with safety confirmations.

## Design Principles

**Teaching Through Descriptions**: Rather than writing "limit: int - Number of results," compose educational descriptions explaining defaults, constraints, and usage patterns.

**Type Hints as Documentation**: Using Literal["asc", "desc"] instead of generic str tells the AI exactly which values are valid.

**Smart Error Responses**: Error messages should guide rather than reject. Include hints explaining what went wrong and how to correct the approach.

**Intelligent Defaults**: Make the most common use case work with minimal parameters.

## Why Six Works

- Context Efficiency — reduces cognitive overhead
- OpenAI Alignment — follows ChatGPT's MCP requirements
- Parameter Composition — one tool with eight parameters consumes less context than eight separate tools
- Semantic Clarity — unified operations match actual user intent patterns
