Model Context Protocol Overview

MCP is an open JSON-RPC 2.0 protocol connecting LLM applications to external tools, data, and services through a host-client-server architecture with capability negotiation.
mcpprotocolarchitecture

Overview

The Model Context Protocol (MCP) is an open protocol that standardizes how LLM applications integrate with external data sources, tools, and services. Originally created by Anthropic and released in November 2024, it was donated to the Agentic AI Foundation (AAIF) under the Linux Foundation.

MCP draws inspiration from the Language Server Protocol (LSP), which standardized how IDEs interact with programming language tooling. MCP does the same for AI applications — it defines a universal way to connect LLMs with the context they need.

How It Works

MCP follows a client-host-server architecture. The host (e.g., Claude Desktop) creates client instances that each maintain a 1:1 session with a server. Servers expose resources, tools, and prompts via MCP primitives. Communication uses JSON-RPC 2.0 over stdio or Streamable HTTP.

Why was Model Context Protocol created and what problem does it solve?
MCP standardizes how LLM applications connect to external tools and data, eliminating per-integration glue code. It provides a universal protocol (like LSP did for IDEs) so any host can connect to any server through capability negotiation.
